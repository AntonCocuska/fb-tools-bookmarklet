FB Tools

Одна закладка, семь вкладок: Доступ (отправка/приём запросов), Rename (ID кабинетов → копировать/переименовать), Kill (удаление объявлений / зачистка кабинета), Billing (массовый адрес + привязка карт + основная карта), Каталог (создание каталогов), Права (раздача прав людям/системникам), Партнёрство (расшаривание активов партнёрскому BM).
🔨 FB Tools 🧩 Панелька Ленина
Перетащи нужную закладку на панель. Панелька Ленина — работает на страницах Ads Manager, с группировкой по BM.

Вкладка «Доступ»

  1. Отправка (главный профиль): business.facebook.com/.../settings/ad_accounts → business_id, список ID, «Старт». На лимите 2 pending ждёт приёма и продолжает.
  2. Приём (профиль-владелец): adsmanager.facebook.com → тот же список, «Старт» — принимает по кругу.
  3. Если режим не совпадает со страницей — закладка перекинет на нужный сайт; после загрузки нажми её ещё раз.

Вкладка «Rename»

  1. adsmanager.facebook.com/adsmanager/ → «Загрузить аккаунты».
  2. «Копировать все / выделенные» (ID в буфер) или «Переименовать → ID».

Вкладка «Kill»

  1. Открой adsmanager.facebook.com/adsmanager/ с выбранным кабинетом (act=... в URL).
  2. Kill Ads: выдели объявления (в URL появится selected_ad_ids=...) → кнопка «Kill Ads».
  3. Kill Account: введи номер кабинета для подтверждения → «Kill Account» (стирает все кампании/адсеты/объявления).

Вкладка «Billing»

  1. Работает там, где открыт твой Billing Hub — adsmanager.facebook.com или business.facebook.com (worker-вкладка открывается на том же домене).
  2. Set address: список Account IDs → массово ставит адрес/валюту/таймзону (US/DE/Wilmington/19801, USD, Europe/Kiev).
  3. Attach card: Account IDs ↔ Cards построчно, опц. «сделать карту основной».

Вкладка «Каталог»

  1. Токен: со страницы или свой (системный юзер BM с catalog_management + business_management).
  2. Укажи BM ID, опц. Pixel ID, и список строк: имя + Tab + feed URL (feed необязателен) — можно вставить прямо из таблицы.
  3. «Создать каталоги»: каталог → feed + upload → пиксель → ручной товар → product sets Set (по фиду) и Start.
  4. Если Set = PENDING (фид ещё не проиндексирован) — позже жми «Досоздать PENDING Set». «Проверить feed» показывает статусы загрузок.
  5. «Копировать результат» — TSV со всеми ID для вставки обратно в таблицу.

Вкладка «Права»

  1. Работает на fb_dtsg (куки сессии) — токен не нужен. Выбери BM из списка или впиши BM ID.
  2. «Загрузить людей и активы»: слева — люди и системные пользователи, справа — активы (кабинеты, страницы, пиксели, каталоги) с группировкой и пагинацией.
  3. Поиск над активами — фильтрует по id или названию. Отмечай галочками по одному, по группе или «выделить все».
  4. «Раздать максимальные права» — каждому выбранному человеку на каждый актив выдаются максимально доступные права (лучший зонтик + вложенные таски, с перебором уровней при отказе).

Вкладка «Партнёрство»

  1. Тоже на fb_dtsg, токен не нужен. Расшаривает активы твоего BM партнёрскому BM.
  2. Укажи свой BM ID и BM партнёра, жми «Загрузить активы». Поиск по id/названию работает так же.
  3. Отметь активы и «Расшарить партнёру» — с максимально доступными правами (для делегированных страниц берётся уровень, который партнёру реально можно выдать).
Kill необратим. Удалённые объекты не восстанавливаются. Kill Account запускай только на кабинете, который реально хочешь обнулить.
Приём — «чувствительное действие»: первый раз FB может попросить код из почты (прими один запрос вручную).
Токены (Rename/Kill) и Billing работают с куками сессии на нужном поддомене. Коды: 190 — токен истёк; 200/10 — нет ads_management; 4/17 — rate-limit; 31 — пройди аутентификацию кабинета в UI.
